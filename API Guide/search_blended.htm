Blended Searches

Blended Searches via XML

With a single query, developers can retrieve results across all the different product categories, sorted by relevance.  For a blended search, you pass in search keywords but no mode or page parameter.  This will return up to 3 results for each of the product categories currently available.  This can currently mean up to 45 results for a single query.

Please note: Amazon Web Services currently offers fifteen product categories for a blended search, though this number will likely change in the future.  You’ll want to write your code to process a return value of any size instead of hard-coding a specific result size.

Call and Response

Request Format

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]&BlendedSearch=[search query goes here]&type=[lite or heavy]&f=xml

Response

The Blended Search request returns up to 3 results for each of the 15 different product categories.

Example

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer's token goes here]&BlendedSearch=garden&type=lite&f=xml
