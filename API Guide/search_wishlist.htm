Wish List and Registry Searches

Wish List and Registry Searches via XML

As with Listmania! lists, to call up a specific wishlist or registry in XML, you must first find the appropriate ID number. Simply travel to the page that contains the list that you are interested in, and look for the list's 13 character ID in web page's URL. (It appears after the "/list-browse/-/" string). For example, in the following URL the list ID would be "19J6Y001ZYYD3". 

http://www.amazon.com/exec/obidos/registry/19J6Y001ZYYD3

Once you find the ID, you simply plug it into the request format below.

Call and Response

Request Format

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]&WishlistSearch=[wishlist ID goes here]&type=[lite or heavy]&f=xml

Response

The WishlistSearch request returns a ProductInfo node. The ProductInfo node contains an array of Detail nodes.

Example

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]&WishlistSearch= 19J6Y001ZYYD3&type=lite&f=xml

Wish List and Registry Searches via SOAP

This section shows how to create a request for wish list and registry searches via SOAP.  This request will return a SOAP object containing the Lite or the Heavy form of the data.

Call and Response

Request Name

WishlistSearchRequest

Request Format

wishlist_id  string

page  string

mode  string

tag  string

type  string

devtag  string

Response

The WishlistSearch request returns a ProductInfo node. The ProductInfo node contains an array of Detail nodes.
