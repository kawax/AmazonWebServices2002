Price Range Searches

AWS 3.0 features a new parameter that you can use to generate search results based on a price range. By adding this parameter to a search request--say, a search for new DVDs--you can narrow the results to fit a high-to-low price range. You can also use this search to find products at a specific price point (for example, a search for books that cost $7.99). This search will work with all searches except power search, similarities search, listmania search, wishlist search, and seller searches.

Please note: The prices returned in a price range search are in US currency pennies, so the given search of price=2000-5000 would find items priced at between 20 and 50 dollars.

Example

Here's an example of a standard browse node request:

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[Developer’s token goes here]&BrowseNodeSearch=[browse node goes here]&mode=[product line goes here]&type=[lite or heavy] &page=[page # goes here]&f=xml

By adding the price range parameter to this request, you can generate results to fit that price range.

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[Developer’s token goes here]&BrowseNodeSearch=[browse node goes here]&mode=[product line goes here]&type=[lite or heavy] &page=[page # goes here]&price=[price range goes here]&f=xml

Example

To generate a "heavy" XML document listing top-selling books, the URL would look like this:

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]&BrowseNodeSearch=1000&mode=books&type=heavy&page=1&price=20-50&f=xml
