Getting Started

What Are Amazon Web Services for Sellers?

Amazon Web Services for Sellers provides a secure and automated method to upload inventory details to Amazon.com (zShops and Amazon Marketplace) on a scheduled basis. It also enables the automation of other important seller activities such as generating order reports and issuing bulk refunds. This is a first step toward real-time updates that will enable selling partners to better manage their inventory in multiple physical and virtual locations.

This project utilizes HTTPS (Hypertext Transfer Protocol Secure), a protocol that is designed to transfer encrypted information between computers over the World Wide Web. HTTPS is a unique protocol that combines SSL and HTTP to provide secure data transmission. The goal of this document is to help our valued partners understand the technical requirements of our HTTPS solution.

Getting Registered

In order to sell products on the Amazon Web site, you’ll first need to register with Amazon.com.  To get started, see the help information contained on the Amazon Web site.

Security

Use of HTTPS ensures that all data uploads and downloads will be encrypted and secure.

Sellers will need to use SSL (Secure Sockets Layer) and PKI (Public Key Infrastructure) technology to communicate securely with Amazon.com—if you are using Perl, note that both of these technologies are built into Perl to ensure that scripts written to automate the data communication are correctly configured.

The identity of the seller will be verified using the "Basic HTTP Authentication Scheme." The seller will provide his or her e-mail address and password in the HTTPS header, encoded using base64. 

Automation

Please note that the below instructions are detailed for partners using Perl. However, you may use any language.

Script Configuration

In order to automate the secure uploading and retrieval of files and/or reports, selling partners will need to create an HTTPS client. This client will consist of scripts that automate the uploading and retrieval of files.

  • Upload: To upload a file, the seller uses a Perl script that they have written to send a POST request with the file attached in the request body. The seller should include both their Amazon.com e-mail address and password in the request header.
  • Retrieve: To retrieve a file, the seller sends a POST request for the particular file. Amazon.com sends the requested file in an HTTPS response, at which point the seller would redirect the response to a specified directory and then save the file.
  • Notifications: Selling partners who participate in the HTTPS solution can have the automatic e-mail notifications from Amazon.com regarding uploads and reports turned off, relying instead on the HTTPS responses in XML.

If an upload fails due to internal error, Amazon.com will send e-mail with the batch status and information.
