Generating Additional Product Results

Because some queries could potentially return a large number of results, Amazon breaks down the results and returns them in ten-item chunks.  To obtain more than ten results for a search, you simply need to pass a "page" parameter in your URL. For example, to get the top 30 books written by Stephen King (instead of simply the top 10), you would need to make three separate calls into Amazon.com, with the parameters page=1 (which will give you books #1 through #10), page=2 (books #11 through #20) and page=3 (books #21 through #30).

Specifying a page value of 1 requests the first page, 2 the second, and so forth. A special value called TotalResults is returned as a part of any request that can return more than one result. This value indicates the total number of items (not pages) that are available from the query.  The total number of pages that are available from the query will be returned in a value called TotalPages.

The process for obtaining “N” items in a search is to create an algorithm as shown:

  1. Form the basic query (omitting the page parameter)
  2. Set a page variable to 1
  3. Finalize the query by inserting the page variable into the basic query
  4. Issue the query
  5. Repeat steps 4 and 5 until the desired number of results have been fetched
