Accessing Amazon.com Data via SOAP

This section explains how to access Amazon.com data using SOAP.

Amazon.com SOAP Usage Overview

SOAP lets third-party developers retrieve Amazon catalog information using remote procedure calls. This information is encoded using XML, although this is transparent to you in the course of normal usage. Please see our SOAP WSDL for more information on the contents of AWS SOAP responses.

Guidelines for Accessing Amazon.com Data via SOAP

This section shows general guidelines for accessing Amazon.com data via SOAP.

  1. Your Web site or application should make no more than one (1) request per second to our servers.
  2. Your Web site or application should have some kind of caching mechanism to temporarily store Amazon catalog data.
  3. Your Web site or application should not cache Amazon data for more than twenty four (24) hours.
  4. Your Web site or application should contain some measure of error handling.
  5. Your Web site or application should not be configured to immediately retry failed server requests—doing so only exacerbates problems.
