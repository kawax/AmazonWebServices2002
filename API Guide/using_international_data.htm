International Browse Nodes

The browse structure is the way that Amazon.com organizes products into subjects and categories on our site so that customers can find the product they want easily and efficiently. A browse ID is a number that corresponds to a general subject area of Amazon.com. Browse nodes are required for certain AWS searches.

Our international Web sites also use browse nodes, although each international site has a different set of nodes, even when their products are organized in a similar way to the US catalog. For example, the node for comedy DVDs is “163357” (for the US) and “501866” (for the UK):

  • http://www.amazon.com/exec/obidos/tg/browse/-/163357
  • http://www.amazon.co.uk/exec/obidos/tg/browse/-/501866

Be sure to use the correct browse node when querying our international services. We list common international browse nodes in this section.


International ASINs

An ASIN, or “Amazon Standard Item Number”, is the unique ID code for a product in Amazon’s catalog. It is also a common parameter used in certain AWS searches. All of our international sites use ASINs, although the ASIN will differ on a country-by-country basis, even if it is the same product in both catalogs. For example, the “Indiana Jones 4-pack DVD set” is represented by two different ASINs – “B00003CXC5” (for the US) and “B00009ZWAB” (for the UK):

  • http://www.amazon.com/exec/obidos/ASIN/B00003CXC5
  • http://www.amazon.co.uk/exec/obidos/ASIN/B00009ZWAB
Be sure to use the correct ASIN number when querying our international services.


Using Associates IDs in International Service Calls

If you want to get credit for sales that you generate through a particular country’s AWS links, you will need to be a member of that country’s Associates program (and embed your Associates ID in your calls to the service). For more information about the UK, DE and JP Associate programs, or to sign up, please see:

  • http://www.amazon.co.uk/associates
  • http://www.amazon.de/exec/obidos/tg/stores/static/-/associates/associates/
  • http://www.amazon.co.jp/exec/obidos/subst/associates/join/associates.html

Please note that the Amazon.de and Amazon.co.jp Associate programs are only supported in the German and Japanese languages, respectively.

IV. Sample Calls


XML/HTTP Call Format (UK)

To target a given XML/HTTP call to Amazon.co.uk, change the URL from xml.amazon.com... to xml-eu.amazon.com..., and pass the locale parameter, as shown in the following request:

http://xml-eu.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]& type=[lite or heavy]&mode=[product line goes here]&page=[page # goes here]&f=xml&KeywordSearch=[subject keyword goes here]&locale=uk


XML/HTTP Call Format (DE)

To target a given XML/HTTP call to Amazon.de, change the URL from xml.amazon.com... to xml-eu.amazon.com..., and pass the locale parameter, as shown in the following request:

http://xml-eu.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]& type=[lite or heavy]&mode=[product line goes here]&page=[page # goes here]&f=xml&KeywordSearch=[subject keyword goes here]&locale=de


XML/HTTP Call Format (JP)

To target a given XML/HTTP call to Amazon.co.jp, pass the locale parameter, as shown in the following request. (You do not need to change the URL).

http://xml.amazon.com/onca/xml3?t=webservices-20&dev-t=[developer’s token goes here]& type=[lite or heavy]&mode=[product line goes here]&page=[page # goes here]&f=xml&KeywordSearch=[subject keyword goes here]&locale=jp


SOAP Call Format

To target a given SOAP call to Amazon.co.uk, Amazon.de or Amazon.co.jp, you will need to add a locale string parameter to your request (“uk”, “de” or “jp”). For the UK and Germany, you will also need to change your SOAP WSDL from http://soap.amazon.com... to http://soap-eu.amazon.com.

V. Working with Japanese Characters

Amazon.co.jp Web Services are UTF-8 encoded, so you cannot enter pure Japanese characters in your XML or SOAP service calls. Instead, you must convert Japanese characters to UTF-8 before passing requests to our services.

Similarly, the XML that we pass back is UTF-8 encoded as well. This may not be ideal for certain applications where the information is ultimately displayed on a Web page. If this is the case, you will need to convert the data to a format that works best for your audience (such as shift_jis).

One more thing: You may encounter some “invalid character errors” when attempting to view the XML in Internet Explorer. If you do run into this problem, you may want to either use a different browser (such as Mozilla which doesn’t seem to have the same problems), or an XML client like XMLSpy.
