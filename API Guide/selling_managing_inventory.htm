Types of Requests (Detailed)

Sellers will be able to manage their inventory at Amazon.com using the following request types:

  • Add/Modify/Delete
  • Modify/Delete
  • Purge and Replace

The difference between each of these request types is explained below.

Inventory Upload via Add/Modify/Delete

Use Case: A seller should use the Add/Modify/Delete request to upload a batch of Amazon Marketplace and/or zShops listings containing additions, modifications, and/or deletions to their current inventory. Deletions must be marked as such in their template, (with a d in the add-delete field) and our system will update changed items to reflect the modifications.

A seller should choose the Add/Modify/Delete option when they have a large amount of inventory to upload onto the Amazon.com platform.

HTTPS POST request to the following URL:

https://secure.amazon.com/exec/panama/seller-admin/catalog-upload/add-modify-delete

Header Content:

  • Username (using basic authentication)
  • Password (using basic authentication)
  • Developer Token
  • Choice of program type
    • Amazon Marketplace only (for UIEE users) (default)
    • zShops only
    • Amazon Marketplace and/or zShops
  • Choice of file format
    • Tab-delimited (default)
    • UIEE

Message Body:

The body of the message should include the actual inventory file desired for upload with the required Amazon.com headers and values. These required values can be found via the URL below for program type "zShops only":

http://s1.amazon.com/exec/varzea/subst/seller-admin/fixed-price-template-fields.html

and via the URL below for program type "Amazon Marketplace and/or zShops":

http://s1.amazon.com/exec/varzea/subst/seller-admin/marketplace-template-fields.html

Inventory Upload via Modify/Delete

Use Case: A seller should use the Modify/Delete request when they want to upload modifications to their current inventory but don't want to supply all of the fields that an Add/Modify/Delete file requires. The field "sku" is required in this upload file, as well as at least one additional field that they are choosing to modify. The allowable fields of modification in the Modify/Delete Only option are price and quantity. Please note: to submit a delete, use the quantity column and enter "0" as the value.

HTTPS POST request to the following URL:

https://secure.amazon.com/exec/panama/seller-admin/catalog-upload/modify-only

Header Content:

  • Username (using basic authentication)
  • Password (using basic authentication)
  • Developer Token
  • Choice of program type
    • Amazon Marketplace only (for UIEE users) (default)
    • zShops only
  • Amazon Marketplace and/or zShops
  • Choice of file format
    • Tab-delimited (default)
    • UIEE

Inventory Upload via Purge and Replace

Use Case: A seller should use the Purge and Replace request when they want to completely purge all of their current Amazon Marketplace and/or zShops listings and replace them with a new set of offerings contained in their template.

HTTPS POST request to the following URL:

https://secure.amazon.com/exec/panama/seller-admin/catalog-upload/purge-replace

Header Content:

  • Username (using basic authentication)
  • Password (using basic authentication)
  • Developer Token
  • Choice of program type
    • Amazon Marketplace only (for UIEE users) (default)
    • zShops only
  • Amazon Marketplace and/or zShops
  • Choice of file format
    • Tab-delimited (default)
    • UIE

Message Body:

The body of the message should include the actual inventory file desired for upload with the required Amazon.com headers and values. These required values can be found via the URL below for program type "zShops only":

http://s1.amazon.com/exec/varzea/subst/seller-admin/fixed-price-template-fields.html

and via the URL below for program type "Amazon Marketplace and/or zShops":

http://s1.amazon.com/exec/varzea/subst/seller-admin/marketplace-template-fields.html
