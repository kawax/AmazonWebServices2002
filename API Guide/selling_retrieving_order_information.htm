Types of Requests (Detailed)

Sellers can also make HTTPS requests to retrieve order information. The two types of requests are detailed below.

Checking Status of Order Fulfillment Reports

Use Case: A seller should use the Report Status Request when they want to check on the status of multiple order reports. They should include the number of reports that they want to check on in the request header.

HTTPS POST request to the following URL:

https://secure.amazon.com/exec/panama/seller-admin/manual-reports/get-report-status

Header Content:

  • Username (using basic authentication)
  • Password (using basic authentication)
  • Developer Token
  • Number of Reports (default = 10)
  • Choice of Report Name
    • Order (default)
    • Preorder
    • Batch Refund
    • Bulk Refund

Retrieving Order Fulfillment Reports

Use Case: A seller should use the Order Fulfillment Request when they want to download a specific report. The request header must include either the ID of the report in question, or a generic report name (Order, Preorder, etc.). If a generic report name is used, then the system will retrieve the latest available report.

HTTPS POST request to the following URL:

https://secure.amazon.com/exec/panama/seller-admin/download/report

Header Content:

  • Username (using basic authentication)
  • Password (using basic authentication)
  • Developer Token
  • Number of Reports (default = 10)
  • Choice of Report Name
    • Order (default)
    • Preorder
    • Batch Refund
    • Bulk Refund
